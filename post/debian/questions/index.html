apt 安装遇到权限问题

描述:

N: 由于文件’/home/eayyoung/Downloads/software/youdao-dict_6.0.0-ubuntu-amd64.deb’无法被用户’_apt’访问，已脱离沙盒并提权为根用户来进行下载。 - pkgAcquire::Run (13: 权限不够)

解决方案：

这个错误信息表明 apt 工具在尝试访问您提供的软件包文件时没有足够的权限。这可能是因为您没有正确地设置文件权限，或者 apt 工具没有被正确地提权。

以下是一些解决这个问题的方法：

  1. 检查文件权限： 首先，确保您有权限访问该文件。您可以使用 ls -l 命令来检查文件的权限：
    ls -l /home/eayyoung/Downloads/software/youdao-dict_6.0.0-ubuntu-amd64.deb
    如果权限不足，您可以使用 chmod 命令来改变文件的权限：
    chmod u+rw /home/eayyoung/Downloads/software/youdao-dict_6.0.0-ubuntu-amd64.deb
  2. 确保 apt 具有正确权限： 通常情况下，apt 工具应该已经具有足够的权限来安装软件包。如果您遇到权限问题，可能需要重新安装 apt 工具：

这将修复系统的依赖关系，并可能解决权限问题。
3. 以 root 用户执行： 您可以尝试使用 su 命令切换到 root 用户，然后执行 dpkg 命令来安装软件包：

  1. 使用 sudo 提权： 如果您不想切换到 root 用户，您可以使用 sudo 来提权执行 dpkg 命令：
    sudo dpkg -i /home/eayyoung/Downloads/software/youdao-dict_6.0.0-ubuntu-amd64.deb

描述：

目前KDE Plasma 5桌面环境非常稳定，很少有破坏性问题导致无法使用桌面，但是，有时KDE Plasma面板可能会冻结并崩溃，这是一个很大的问题。这会直接导致用户进入不了系统界面，或输入正确的用户密码而无响应。

与Gnome Shell不同，KDE Plasma 5没有真正的内置重启机制供用户利用，因此，如果你正在使用KDE并且想要快速重启破损或崩溃的KDE Plasma面板，则需要创建解决方案，在本文中，我们将讨论如何创建一个重启工具，如果你的KDE Plasma面板崩溃，可以让你摆脱问题的困扰。

方案二

  1. 使用terminal命令重新启动面板

    KDE面板是在Linux PC上运行“plasmashell”过程的一部分，因此，如果你遇到面板无法正常工作的问题，那么让它恢复运行的最简单方法就是处理该过程。

    你需要手动停止它，执行此操作的最佳方法是使用killall命令，因为它会立即停止“plasmashell”运行的每个实例，因此，通过按键盘上的Ctrl+Alt+T或Ctrl+Shift+T打开终端并结束“plasmashell”，参考Linux进程管理：用kill和killall命令杀死进程：

     killall plasmashell

    运行killall命令后，你仍然可以访问终端，从这里开始，从终端重启“plasmashell”进程，请务必在命令末尾添加“&”以将其发送到后台：

     plasmashell &

    一旦运行上面的plasmashell命令，屏幕上就会出现一个新的KDE Plasma 5面板，但是，这个新面板依赖于终端，因为它在内部运行，从这里开始，需要将流程发送到后台，为此，请运行disown命令:

     disown

  2. 创建重启脚本

    打开终端以杀死“plasmashell”过程可能非常繁琐，重启面板的一个好方法是创建一个可以运行的快速重启脚本。

    首先，按Ctrl+Alt+T和键盘上的Ctrl+Shift+T打开终端窗口，从那里，按照下面的分步说明进行操作。

    • 步骤1：使用touch命令在主目录中创建一个标记为“panel-restart”的新空白文件： 

      sudo touch panel-restart

    • 步骤2：使用下面的echo命令将shebang添加到脚本的顶部，此命令至关重要，因为没有它，你的Linux系统将不知道如何处理重启脚本：

      echo '#!/bin/bash' > panel-restart

    • 步骤3：运行另一个echo命令将主代码添加到panel-restart脚本：

      echo 'killall plasmashell;plasmashell &' >> panel-restart

    • 步骤4：使用chmod命令更新重新启动文件的权限，以便它可以作为Linux PC上的程序运行：

      sudo chmod +x panel-restart

    • 步骤5：使用mv命令将panel-restart文件放在/usr/bin/中：

      sudo mv panel-restart /usr/bin/

    参考：Linux mv命令使用示例－移动或重命令文件/目录。

    关闭终端窗口，然后，转到KDE桌面并按Alt+F2打开Plasma快速启动窗口，然后，在窗口中，输入以下命令，立即使用我们刚制作的脚本重新启动KDE Plasma 5面板：

     panel-restart

    运行命令后，你的KDE Plasma 5面板应该可以使用了。
