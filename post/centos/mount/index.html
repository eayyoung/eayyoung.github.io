mount -0 选项参数

在 Linux 中，mount -o 用于指定挂载文件系统时的额外选项，这些选项可以控制文件系统的行为，如权限、性能、安全等。以下是 mount -o 常用选项详解，涵盖各种文件系统（如 ext4、NTFS、FAT32、NFS 等）。

1. 通用挂载选项

选项 作用 示例
rw / ro 挂载为读写（rw）或只读（ro） mount -o rw /dev/sdb1 /mnt
noexec 禁止执行文件（增强安全性） mount -o noexec /dev/sdb1 /mnt
nosuid 禁用 SUID/SGID 权限（防提权） mount -o nosuid /dev/sdb1 /mnt
nodev 禁用设备文件（防恶意设备访问） mount -o nodev /dev/sdb1 /mnt
relatime 优化访问时间更新（默认） mount -o relatime /dev/sdb1 /mnt
noatime 完全不更新访问时间（提升性能） mount -o noatime /dev/sdb1 /mnt
sync / async 同步（sync）或异步（async）写入 mount -o sync /dev/sdb1 /mnt
remount 重新挂载已挂载的文件系统 mount -o remount,rw /

2. 权限控制选项

主要用于 NTFS/FAT32 等非 Linux 文件系统（无原生权限支持）：

选项 作用 示例
umask=<值> 设置默认权限掩码（反向） mount -o umask=022 /dev/sdb1 /mnt
dmask=<值> 仅目录权限掩码 mount -o dmask=022 /dev/sdb1 /mnt
fmask=<值> 仅文件权限掩码 mount -o fmask=133 /dev/sdb1 /mnt
uid=<ID> 设置所有者用户 mount -o uid=1000 /dev/sdb1 /mnt
gid=<ID> 设置所有者组 mount -o gid=1000 /dev/sdb1 /mnt

mount -o选项的umask

在 Linux 中，mount 命令的 umask 选项 用于控制挂载文件系统时文件和目录的默认权限。它通过屏蔽（mask）特定的权限位来设置权限，类似于 chmod 的反向操作。

1. umask 的作用

  • umask 值 是一个八进制数，表示要屏蔽的权限（即不允许的权限）。
  • 计算最终权限的公式：
  • 常见 umask 值：
    • umask=000 → 目录 777（drwxrwxrwx），文件 666（-rw-rw-rw-）
    • umask=022 → 目录 755（drwxr-xr-x），文件 644（-rw-r--r--）
    • umask=077 → 目录 700（drwx------），文件 600（-rw-------）

6. 注意事项

  1. ext4/xfs 等 Linux 文件系统：
    • 通常不需要 umask，权限由文件系统自身管理。
    • 直接使用 chmod 和 chown 更灵活。
  2. NTFS/FAT32：
    • 必须通过挂载选项（umask、uid、gid）控制权限。
  3. umask 是反向掩码：
    • umask=022 实际允许权限是 755（目录）和 644（文件）。
