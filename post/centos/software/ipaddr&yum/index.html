cos-软件管理-ip地址配置&yum源

发表于1970-01-01|centos软件包管理
|阅读量:
  1. setup 工具: 文本模式设置工具（配置固定IP地址）

  • 网络配置 >> DNS配置 >> 选择网卡 >> 进行配置

  • [root@localhost~]# vi /etc/sysconfig/network-scripts/ifcfg-eth0
    #把ONBOOT=”no” 改为 ONBOOT=“yes” 启动网卡

  • [root@localhost~]# service network restart //重启网络服务

  1. VMware 工具: liunx配置固定IP

  • vmware虚拟网络编辑器 >> vmnet8 Nat主机模式更改设置>> 子网IP和子网掩码修改 >> nat设置>>网关设置

    （子网：192.168.66.0 掩码：255.255.255.0 网关：192.168.66.66）

  • [root@localhost~]# vi /etc/sysconfig/network-scripts/ifcfg-eth0
    #把BOOTPROTO=”dhcp” 改为 ONBOOT=“static”（boot prototcol ：引导协议 dhcp：动态协议 static：静态协议）

    • 并在末尾新增内容：

    • IPADDR：”192.168.66.88” ip地址

    • NETMASK: “255.255.255.0” 子网掩码

    • GATEWAY: “192.168.66.66” 网关

    • DNS1: “192.168.66.66” 局域网，写网关即可

  • [root@localhost~]# service network restart 重启网络服务

  1. 网络yum源 :

  • [root@localhost~yum.repos.d]# vi /etc/yum.repos.d/CentOS-Base.repo

  • 标识说明：

[base] 容器名称，一定要放在[ ]中
name 容器说明，可以随意填写
mirrorlist 镜像站点，这个可以注释掉
baseurl 我们的yum源服务器地址。默认时CentOS 官方的yum源服务器，是可以使用的，可以更改自己喜欢的yum源地址。
enabled 此容器是否生效，如果不写或写成enable=q1都是生效的，enable=0就是不生效的
gpgcheck 如果是1是指RPM的数字证书生效，是0则不生效
gpgkey 数字证书的公钥文件保存位置，不用修改
