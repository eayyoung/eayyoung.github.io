python虚拟环境

发表于2025-02-07|python
|阅读量:

在 Linux 系统中，创建和管理 Python 虚拟环境是隔离项目依赖的推荐做法。以下是详细步骤和配置方法：

1. 安装 Python 虚拟环境工具

  • Python 3 内置 venv 模块：无需额外安装。

  • Debian/Ubuntu 系统安装：

    sudo apt update
sudo apt install python3-venv

2. 创建虚拟环境

  • 使用 venv（推荐）：

    cd ~/myproject
python3 -m venv venv

  • 使用 virtualenv（需安装）：

    pip3 install virtualenv
virtualenv venv

3. 激活虚拟环境

  • 激活虚拟环境：

    source venv/bin/activate

    激活后，命令行提示符会显示环境名 (venv)。

4. 配置虚拟环境

  • 安装依赖包：

    pip install numpy jupyter ipykernel

  • 导出依赖列表：

    pip freeze > requirements.txt

  • 从 requirements.txt 安装依赖：

    pip install -r requirements.txt

  • 配置 Jupyter 内核（可选）：

    pip install ipykernel
python -m ipykernel install --user --name=venv --display-name "Python (venv)"

5. 退出虚拟环境

  • 退出虚拟环境：

    deactivate

6. 删除虚拟环境

  • 删除虚拟环境目录：

    rm -rf venv/

7. 常用命令速查

操作 命令
创建环境 python3 -m venv venv
激活环境 source venv/bin/activate
退出环境 deactivate
安装包 pip install package_name
导出依赖列表 pip freeze > requirements.txt
删除环境 rm -rf venv/

8. 虚拟环境管理工具（可选）

  • virtualenvwrapper：简化虚拟环境管理。

    pip install virtualenvwrapper
echo "export WORKON_HOME=$HOME/.virtualenvs" >> ~/.bashrc
echo "source /usr/local/bin/virtualenvwrapper.sh" >> ~/.bashrc
source ~/.bashrc

    使用示例：

    mkvirtualenv myenv      # 创建环境
workon myenv            # 激活环境
deactivate              # 退出环境
rmvirtualenv myenv      # 删除环境

注意事项

  • 权限问题：避免在系统 Python 环境中安装包，始终优先使用虚拟环境。
  • 环境隔离：每个项目应使用独立的虚拟环境。
  • IDE 配置：在 VS Code、PyCharm 等工具中，需手动选择虚拟环境的 Python 解释器路径（如 venv/bin/python3）。

通过上述步骤，你可以轻松创建和管理 Python 虚拟环境，确保项目依赖的独立性和可移植性。希望这些信息对你有所帮助！
