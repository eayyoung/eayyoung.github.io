Hexo异地上传与更新

发表于2025-02-06|hexo
|阅读量:

要实现这个目标，你需要确保Windows和Debian两个系统之间能够共享同一份博客文件，并且能够在Debian上通过VSCode编辑这些文件，然后将它们上传到GitHub。以下是一个详细的步骤指南：

  1. 设置共享文件夹：

    • 首先，在Windows系统中创建一个文件夹来存放Hexo博客文件，例如 C:\\HexoBlog。
    • 在Debian系统中，你可以使用NFS（网络文件系统）或者Samba来访问Windows中的这个文件夹。

  2. 安装并配置NFS或Samba：

    • 如果你选择使用NFS，需要在Windows上安装NFS服务，并在Debian上挂载Windows的共享文件夹。
    • 如果你选择使用Samba，需要在Windows上安装并配置Samba服务器，然后在Debian上挂载Samba共享。

  3. 挂载共享文件夹：

    • 假设你选择了Samba，那么在Debian上执行如下命令来挂载Samba共享文件夹：

      sudo mount -t cifs -o username=your_username,password=your_password //windows_ip/hexo /mnt/hexo

      其中 windows_ip 是你的Windows系统的IP地址，/mnt/hexo 是你在Debian系统中挂载的目录。

  4. 配置VSCode：

    • 打开Debian系统中的VSCode，然后打开挂载的博客文件夹 /mnt/hexo。
    • 确保你已经安装了Git插件，这样可以直接从VSCode中提交代码。

  5. 配置SSH密钥：

    • 在Debian系统中生成SSH密钥对，并将公钥添加到GitHub账户中，以便在不输入密码的情况下进行Git操作。

    • 执行以下命令生成SSH密钥：

      ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

    • 将生成的公钥（~/.ssh/id_rsa.pub）添加到GitHub账户的SSH密钥管理中。

  6. 配置Git仓库：

    • 在Debian系统中打开终端，进入挂载的博客文件夹：

      cd /mnt/hexo

    • 初始化Git仓库（如果还没有初始化的话）：

      git init

    • 添加远程仓库：

      git remote add origin git@github.com:your_username/your_repo.git

  7. 编辑和同步博客文件：

    • 使用VSCode编辑博客文件，保存更改后，可以通过VSCode的Git插件提交更改：

      git add .
git commit -m "Your commit message"
git push origin master

    • 这样就可以将修改后的文件推送到GitHub上的Hexo博客项目中。

  8. 更新Hexo博客：

    • 确保在Debian系统中安装了Hexo，如果没有安装可以执行：

      npm install -g hexo-cli

    • 在博客文件夹中运行Hexo命令来生成静态文件并部署：

      hexo generate
hexo deploy

    补充：如果hexo没有配置全局环境，则使用npx hexo来代替hexo命令。

通过以上的步骤，你可以在Debian系统中编辑和管理Hexo博客文件，并将其同步到GitHub上。确保每个步骤都正确完成，这样可以保证数据的一致性和安全性。
